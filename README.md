# DeliverySystem

## Структура проекта
Models/ - для всех объектов, которые описывают сущности и данные

Patterns/ - я отдельно вынес, чтобы показать все паттерны, которые использовал. Кроме того, в этой папке есть Services - классы, которые выполняют действия и управляют процессами

DeliverySystem.Tests/ - лежат unit-тесты

## Паттерны, которые использовал
Builder - Создание объекта Order сложное - нужно установить клиента, адрес, тип заказа, добавить несколько блюд, рассчитать стоимостьо. OrderBuilder инкапсулирует процесс создания, позволяет создавать разные типы заказов (стандартные, экспресс) с одинаковым интерфейсом

Abstract Factory - Нужно создавать семейства связанных объектов заказов разных типов (стандартные, экспресс, премиум). IOrderFactory предоставляет интерфейс для создания целых семейств заказов. Обеспечивает согласованность создаваемых объектов

State - Заказ проходит через различные состояния (подготовка, доставка, выполнен), поведение зависит от текущего состояния. IOrderState инкапсулирует поведение для каждого состояния. Упрощает добавление новых состояний и делает переходы между ними управляемыми

Strategy - Нужны разные алгоритмы расчета стоимости: стандартные, со скидками, для лояльных клиентов, для экспресс-доставки. Позволяет легко добавлять новые типы расчета без изменения кода заказа.

Facade - даёт простой единый интерфейс для работы с системой (создать заказ, добавить блюда, перейти состояние
